# 会话体验批次计划：向 "K8s 版 Claude 网页端" 靠拢

> 分支：`claude/kagent-chat-ux`（基于 main @ 9d71de3）
> 背景：产品方向是"深度集成 Kubernetes 集群的 Claude 网页版"。
> 本批实现 Claude 网页端已验证的高价值会话交互，纯前端、零后端风险。

---

## 一、本批范围（对照 Claude 网页版）

| # | Claude 有的能力 | kagent 现状 | 本批做法 |
|---|----------------|------------|---------|
| 1 | 每条回复可一键复制 | 只有点赞/点踩 | 助手消息操作行加 CopyButton（复用共享组件） |
| 2 | 用户消息可编辑后重发 | 无 | 用户消息 hover 出现编辑图标 → 文本回填输入框并聚焦（追加式重发，语义诚实：不改历史） |
| 3 | 会话列表搜索 | 只有时间分组 | 侧栏顶部搜索框，按标题即时过滤（三个分组内联过滤） |
| 4 | 会话导出（可读格式） | 仅原始 task JSON | 会话菜单加 "Export Markdown"：角色分节 + 工具调用摘要 + 附件/产物名，`sessionToMarkdown()` 纯函数 + 单测 |

不做（明确排除及原因）：
- **真正的 regenerate/编辑改历史**：需要后端支持截断会话事件，追加式语义下会误导用户，留待后端能力；
- **消息虚拟化**：需引入 react-virtuoso 依赖，独立批次做性能专项；
- **对话分支 fork**：需要后端 session 复制接口。

## 二、后续路线（"K8s 版 Claude" 差距清单，供排期）

| 方向 | 说明 | 依赖 |
|------|------|------|
| 会话内全文搜索 | 跨会话搜消息内容 | 后端搜索接口（DB LIKE / FTS） |
| Regenerate / 编辑改历史 | 截断最后一轮再生成 | 运行时 session 事件删除能力 |
| 对话分支 | 从任意节点 fork 新会话 | session 复制 API |
| 长会话虚拟化 | react-virtuoso | 前端依赖引入 |
| K8s 上下文注入 | @-提及集群资源（pod/deploy）自动附上 describe 结果 | 走 agent 工具预调用或 controller 代理 |
| 集群健康横幅 | 会话页顶部显示集群/agent 状态 | controller 已有部分状态接口 |
| A2 审批规则 + 会话级放行 | "本会话总是允许" | HITL 流程改造（原计划 A2） |

## 三、验收

tsc / eslint / jest（含 sessionToMarkdown 单测）/ 生产构建全绿；每项功能改动独立、可单独回退。
